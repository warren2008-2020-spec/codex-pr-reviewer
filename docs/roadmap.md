# Roadmap

## Near term

- surface changed file summaries in JSON output
- add review note templates for common PR patterns

## Completed

- add realistic fixtures for authentication, dependency, migration, and rename-only pull requests
- publish a JSON Schema and validate repository configuration
- emit file-aware GitHub Actions annotations for review findings
- distinguish lockfile-only changes from dependency manifest changes

## Next

- support diff input instead of directory-only inspection
- add optional heuristics for docs, refactors, and release-only changes
- make risk thresholds configurable

## Later

- GitHub Action mode
- repository-specific review policies
- richer maintainer guidance for Codex-assisted review workflows
